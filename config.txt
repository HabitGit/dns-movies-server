В главной директории проекта, рядом с docker-compose файлом должнен
лежать файл с переменными .env , пример переменных в .env.example

Внутри директории server каждый проект - это отдельное nest приложение.

В каждом приложении три файла для переменных:
.development.env - содержит переменные для разработки без докера
.docker.env - содержит переменные для разработки в докере
.production.env - содержит переменные без значений, файл для фронтэнда,
в который они внесут свои переменные.
Пример переменных в .env.example

Запуск приложений без докера:
npm run start:dev - для .development.env переменных
npm run start:docker - для .docker.env переменных. -- НЕ ЗАПУСКАЕМ, ТОЛЬКО ДЛЯ DOCKER COMPOSE!
npm run start - для .production переменных.

В процессе разработки инструкции будут меняться, с каждым внесенным изменением в этот файл
в поле V - версия пишем изменения что бы следить за ходом изменений.
